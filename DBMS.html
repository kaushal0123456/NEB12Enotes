Understanding Data, Information, and Databases

What is Data?

Data consists of raw, unprocessed facts, figures, or symbols collected from various sources. It represents values or observations that, in isolation, may lack meaning.

Example: A series of numbers such as 85, 90, 78 representing student scores before any analysis.

What is Information?

Information is data that has been processed, organized, or structured to provide context and meaning. It transforms raw data into a useful and interpretable form that supports decision-making.

Example: Calculating "The average score of the class is 84.3" provides insight derived from the raw student scores.

Key Features of Quality Information

  • Accuracy: Must be precise and error-free to ensure reliability.
  • Relevance: Pertinent to the context or the decision at hand.
  • Timeliness: Delivered promptly and kept up-to-date.
  • Completeness: Contains all necessary details without omission.
  • Clarity: Presented in an understandable and well-organized manner.

What is a Database?

A database is a systematically organized collection of related data stored electronically, enabling efficient storage, retrieval, and management of information.

Example: A school’s database might contain student personal details, exam results, and attendance records, all managed cohesively.

Purpose of a Database

The primary objective of a database is to organize data to facilitate efficient storage, quick retrieval, and reliable management. Databases support multiple users and applications while ensuring data security and integrity.

  • Efficient storage and management of vast volumes of data.
  • Fast, accurate, and reliable data retrieval.
  • Facilitation of data sharing across various users and applications.
  • Protection of data through security and access control mechanisms.
  • Reduction of data duplication and inconsistency.

Common Database Terminologies

Table:
A structured arrangement of data organized into rows and columns, representing entities and their attributes.
Field:
A single column within a table, representing a specific attribute or property of an entity.
Record:
A row in a table that contains data for all fields, representing a single instance of the entity.
Tuple:
Another term for a record or row within a relational database table.
Object:
In object-oriented databases, an object is a complex data structure encapsulating both data and behaviors related to a real-world entity.
Keys:
Fields used to uniquely identify records or establish relationships between tables.
  • Primary Key: A unique identifier for each record in a table.
  • Foreign Key: A field in one table that references the primary key in another, establishing relational links.
  • Candidate Key: Fields that qualify as potential primary keys.

Data Dictionary

A data dictionary is a centralized repository that stores metadata describing the database’s structure, including definitions of tables, fields, data types, and constraints. It facilitates understanding and managing the database schema.

Database Management System (DBMS)

Introduction

A DBMS is software that enables users to create, access, manage, and manipulate databases efficiently while enforcing data integrity, security, and consistency.

Objectives

  • Efficient data storage and retrieval mechanisms.
  • Ensuring accuracy and consistency of data across multiple transactions.
  • Providing security through access control and authorization.
  • Supporting concurrent access for multiple users without conflicts.
  • Reducing data redundancy by centralizing data storage.
  • Offering backup and recovery features to safeguard against data loss.

Advantages

  • Improved data sharing and accessibility across users.
  • Enhanced security and privacy controls.
  • Improved data integrity and minimized duplication.
  • Robust backup and recovery capabilities.
  • Centralized data management simplifying administration.

Disadvantages

  • High initial costs for system installation and licensing.
  • Complex system design requiring specialized expertise.
  • Need for trained personnel to maintain and operate.
  • Potential risks of system failures causing data loss if backups are inadequate.

Types of Database Models

Hierarchical Model:
The Hierarchical Model was the first database management system model. This concept uses a hierarchical tree structure to organise the data. The hierarchy begins at the root, which contains root data, and then grows into a tree as child nodes are added to the parent node. This model accurately depicts several real-world relationships such as food recipes, website sitemaps, and so on.
Network Model:
The network model in DBMS is better than the hierarchical model as there are more interrelations between entities. Supports various relationships such as one-to-one, one-to-many, and many-to-many as well. An entity can have various parents or owners. The connected structure results in high performance.
Relational Model:
Stores data in tables (relations) with rows and columns, using keys to define relationships between tables.
Entity-Relationship (ER) Model:
A conceptual design model that uses entities, attributes, and relationships to map the data and its interconnections, primarily used during database design.

Integrity Constraints

Domain Constraint:
Restricts the permissible values for an attribute, such as type, format, or range.
Entity Integrity:
Guarantees that the primary key is unique and never null, ensuring every record is uniquely identifiable.
Referential Integrity:
Ensures that a foreign key in one table either matches a primary key in another table or is null, preserving relationships between tables.
Keys:
Fields designated to uniquely identify records and link data across tables (Primary, Foreign, Candidate Keys).

Normalization

Introduction

Normalization is the process of structuring a database to minimize redundancy and improve data integrity by organizing fields and table relationships logically.

Normal Forms

First Normal Form (1NF):
Ensures each field contains atomic values and that records are unique.
Second Normal Form (2NF):
Achieved when the database is in 1NF and all non-key attributes are fully functionally dependent on the entire primary key.
Third Normal Form (3NF):
Achieved when the database is in 2NF and all attributes depend only on the primary key, eliminating transitive dependencies.

Advantages

  • Minimizes data duplication and redundancy.
  • Improves consistency and accuracy of data.
  • Organizes data logically and efficiently.
  • Simplifies data maintenance and updates.

Disadvantages

  • Complex queries with multiple table joins may be required.
  • Over-normalization can sometimes degrade database performance.

Centralized vs Distributed Databases

Introduction

Centralized databases store all data at a single location, while distributed databases spread data across multiple locations connected via a network.

Advantages

  • Centralized: Easier management, enhanced security, and simpler consistency maintenance.
  • Distributed: Increased reliability due to redundancy, better availability, and faster local data access.

Disadvantages

  • Centralized: Vulnerable to single point of failure and potential performance bottlenecks.
  • Distributed: More complex design and maintenance; challenges with data synchronization.

Database Security

Overview

Database security encompasses the practices and technologies used to protect databases against unauthorized access, misuse, threats, and vulnerabilities.

Challenges

  • Preventing unauthorized users from accessing sensitive data.
  • Ensuring confidentiality and privacy of stored information.
  • Protecting against threats like SQL injection and cyber attacks.
  • Properly managing user roles, permissions, and authentication.

Security Measures

  • Authentication and authorization controls.
  • Encryption of data both at rest and during transmission.
  • Regular backups and maintaining audit trails.
  • Deployment of firewalls and intrusion detection systems.
  • Role-based access control (RBAC) to restrict user privileges.

Role of the Database Administrator (DBA)

  • Managing user accounts and permissions.
  • Implementing and enforcing security policies.
  • Monitoring database activity to detect suspicious behavior.
  • Planning and executing backup and recovery operations.
  • Applying updates and security patches regularly.

Practical Topics: DDL and DML

Data Definition Language (DDL) commands define or modify the structure of database objects (e.g., CREATE, ALTER, DROP). Data Manipulation Language (DML) commands manage the data itself (e.g., SELECT, INSERT, UPDATE, DELETE).

SQL Data Types

CHAR
Fixed-length character string; length must be specified.
VARCHAR
Variable-length character string; length varies up to a defined limit.
INT (INTEGER)
Integer numerical data type.
FLOAT / REAL
Floating-point numeric data type for approximate values.
DATE
Date values (year, month, day).
BOOLEAN
Logical data type representing TRUE or FALSE.
